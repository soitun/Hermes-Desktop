# Hermes Desktop v2.5.9

Release date: 2026-05-30

## Highlights

- **Release hygiene pass:** audited release-facing source, docs, manifests, and tests for local-machine leftovers before cutting the release.
- **No user session data shipped:** the portable release path still publishes from the WinUI output folder and bundled `skills/`; `%LOCALAPPDATA%\hermes` config, memory, transcripts, and logs remain outside the release artifact.
- **Local-path cleanup:** removed a machine-specific session-title fixture from tests and replaced the alternate package manifest's local-test publisher with the repo's package publisher subject.
- **Chat wiring verification:** documented the Chat runtime path from send/stop/model/session controls through `HermesChatService`, `Agent.StreamChatAsync`, transcript storage, and panel refresh.
- **Skills wiring verification:** documented and fixed install paths for raw `SKILL.md` URLs, GitHub blob URLs, and GitHub repo paths, including one-level category folders and installed frontmatter preservation.
- **Hermes Desktop branding:** the chat welcome block and release-facing docs now refer to Hermes Desktop, not the upstream Hermes Agent UI.

## Verification

- CodeGraph index was current for `C:\Hermes-desktop\Hermes-Desktop-main\Hermes-Desktop-main`.
- `scripts\code-quality.ps1` passed: analyzers, build, and full test suite.
- Desktop tests passed with 750 total tests.
- WinUI app launch was verified after the wiring pass, including a real streamed chat response displayed in the Chat page.

## Versioning

- `Directory.Build.props` now carries `HermesProductVersion` **2.5.9**.

   ```bash
   git push origin v2.5.9
   ```

3. The portable release workflow should publish `HermesDesktop-portable-x64.zip` and its `.sha256` sidecar.
4. After the GitHub Release is live, submit the generated winget manifests for `VyreVaultStudios.HermesDesktop` version `2.5.9`.
