# Hermes Desktop v2.5.6

**Release date:** 2026-05-09
**Portable asset:** `HermesDesktop-portable-x64.zip` (from [GitHub Releases](https://github.com/RedWoodOG/Hermes-Desktop/releases))

## Highlights

### Reference-runtime improvements for dogfooding

This release folds in concrete behavior learned from the downloaded OpenManus and DeepSeek-TUI reference projects, focused on practical Hermes improvements rather than speculative architecture:

- **Planning tool:** adds a first-class plan/task-status tool for agent-visible execution planning.
- **Command registry:** centralizes command metadata and aliases for safer command handling.
- **Streaming accumulator:** keeps incremental assistant text display stable while preserving persisted response text.
- **Structured runtime events:** surfaces token, thinking, tool-status, error, and completion events to the desktop layer.
- **Browser state reporting:** adds a browser state action for inspecting current browser/session state.
- **Large output routing:** redacts secrets before summary/artifact routing and prevents oversized tool output from flooding the chat loop.
- **Post-edit diagnostics:** adds a conservative hook to append diagnostics after edit-style tool results.
- **Durable timeline:** records thread, turn, item, and tool lifecycle data for real execution traces, including successful, failed, denied, and redacted tool results.

### MCP config compatibility

`mcp.json` loading now accepts standard camelCase `mcpServers` keys by using case-insensitive MCP config deserialization. This fixes config files that follow common MCP examples instead of the C# property casing.

### Engineering

- **Tests:** added coverage for planning, command registry behavior, streaming accumulation, browser state formatting, large output redaction/routing, post-edit diagnostics, timeline persistence, agent tool lifecycle timeline records, and MCP camelCase config loading.
- **Verification:** desktop test suite and x64 desktop build passed before release prep.
- **Versioning:** `HermesDesktop` assembly, `Package.appxmanifest`, and appinstaller metadata bumped to **2.5.6.0**.

## Upgrade notes

- **Portable zip:** quit Hermes Desktop, extract the new zip over (or beside) the old folder; `%LOCALAPPDATA%\hermes` is preserved.
- **Dogfood focus:** exercise normal chat, tool calls, denied permissions, large tool outputs, browser state, and MCP startup with a camelCase `mcp.json`.
