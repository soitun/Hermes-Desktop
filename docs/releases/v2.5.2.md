# Hermes Desktop v2.5.2

**Release date:** 2026-04-28  
**Portable asset:** `HermesDesktop-portable-x64.zip` (from [GitHub Releases](https://github.com/RedWoodOG/Hermes-Desktop/releases))

## Highlights

### Windows Sandbox execution

- Added a native Windows Sandbox backend using Windows' built-in `WindowsSandbox.exe` and generated `.wsb` configs.
- Added `Windows Sandbox` to the Desktop execution backend picker.
- The Bash tool now honors the configured terminal backend instead of always defaulting to local execution.
- Sandbox runs map a controlled workspace into `C:\HermesWorkspace`, run the requested command, capture output, and shut down the sandbox.
- If Windows Sandbox is not enabled on the host, Hermes reports a clear setup message instead of falling back to Docker, WSL, or local execution.

### Engineering

- Added `WindowsSandboxBackend` to the execution backend factory.
- Added coverage for generated Windows Sandbox configuration XML.
- Updated architecture docs to replace the old Windows "no sandbox" limitation with the new Windows Sandbox path.
- Versioning: `HermesDesktop` assembly and `Package.appxmanifest` identity set to **2.5.2.0**.

## Upgrade notes

- Existing users do not need to change configuration.
- To use this backend, enable the Windows optional feature **Windows Sandbox** and restart Windows.
- Select **Settings -> Execution Environment -> Windows Sandbox**, then restart Hermes so the configured Bash backend is recreated.
